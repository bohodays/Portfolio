    🚨 이슈 사항

    📌 3D 모델링의 용량 문제로 인한 렌더링 이슈 발생

    3D 모델 파일들의 용량이 작다고 생각했지만(2~3MB 이하) 실제로 렌더링해보니 렌더링 속도가 많이 떨어지고 오류가 발생했습니다. 그래서 gltf-pipeline 라이브러리를 활용해서 3D 모델 파일들을 1MB 이하로 압축했습니다. 그리고 렌더링에 부담이 가는 3D 모델 파일의 효과를 최소화했습니다.

    정원에 꽃을 심는 과정에서 꽃을 클릭하면 해당 꽃이 마우스를 따라다니도록 구현하였는데, 이 과정에서 frame 단위로 마우스 위치를 포착하다보니 렌더링 이슈가 발생했습니다. 그래서 React.memo()를 활용하여 변화가 없는 컴포넌트들의 재렌더링을 방지하여 렌더링을 최적화했습니다.

    📌 일기 작성 기능 구현 시 router를 이동하며 정보를 저장하는 과정에서 이미지 파일 저장 이슈 발생

    하나의 일기를 작성하기 위해서는 많은 로직을 거쳐야 했기 때문에 router를 구분하며 유저를 이동시켰습니다. 여러 router를 거치며 중간 정보를 Redux로 상태 관리하였고, 최종 페이지에서 저장된 정보로 서버에 요청을 보내도록 코드를 구성했습니다.

    위와 같은 구성으로 구현했지만 텍스트 작성 페이지에서 유저가 이미지를 첨부하는 과정에서 문제가 발생했습니다. 저희 서비스는 Amazon의 s3를 활용하여 이미지를 저장하고 불러와서 사용했습니다. 유저가 첨부한 이미지를 Redux에 저장하려고 했지만 직렬화(serialized)된 데이터만 저장할 수 있었기 때문에 저장할 수 없었습니다. 그래서 유저가 이미지를 첨부하면 base 64 형태로 로컬스토리지에 저장한 후 최종적으로 서버에 요청을 보내는 시점에 저장한 정보를 활용하고 로컬스토리지에서 정보를 삭제하는 방식으로 해결했습니다.

    📌 새로고침 시 페이지의 데이터가 사라지는 이슈 발생

    Redux와 Redux Toolkit을 활용하여 데이터들을 상태 관리하였지만 페이지 새로고침 시 관리하고 있는 데이터가 사라지는 이슈가 발생했습니다. 그래서 최상단 App.tsx 컴포넌트에서 Redux에서 관리하는 데이터가 비어있으면 로컬스토리지에 저장되어 있는 access token과 refresh token을 활용하여 유저의 정보를 다시 요청해서 저장하도록 했습니다.
